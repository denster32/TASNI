# TASNI Publication Checklist

**Date:** February 4, 2026
**Purpose:** Ensure the TASNI paper meets publication standards for peer-reviewed journals
**Status:** Phase 7 - Publication Readiness

## Overview

This checklist provides a comprehensive guide for preparing the TASNI paper for submission to a peer-reviewed journal. Following this checklist will ensure the paper meets the highest standards of scientific rigor and reproducibility.

## Pre-Submission Checklist

### Title and Abstract

- [ ] Title is concise and accurately reflects the paper's content
- [ ] Abstract is ≤ 250 words
- [ ] Abstract clearly states the research question
- [ ] Abstract summarizes the methods
- [ ] Abstract reports key findings with uncertainties
- [ ] Abstract states the main conclusions
- [ ] Abstract avoids overstated claims

### Introduction

- [ ] Provides clear motivation for the research
- [ ] Reviews relevant literature comprehensively
- [ ] States the research question explicitly
- [ ] States testable hypotheses
- [ ] Outlines the paper structure
- [ ] References are up-to-date (within last 5 years where possible)
- [ ] References are properly formatted

### Methods

- [ ] Data sources are clearly described with version numbers
- [ ] Selection criteria are justified theoretically
- [ ] Sensitivity analysis is reported
- [ ] Pipeline implementation is described in detail
- [ ] Parallax fitting methodology is documented
- [ ] Temperature estimation methods are described
- [ ] Variability analysis methods are described
- [ ] Statistical methods are described with references
- [ ] ML pipeline is documented with training/validation split
- [ ] All assumptions are stated

### Results

- [ ] All results are presented with uncertainties
- [ ] P-values are reported with confidence intervals
- [ ] Figures are clear and informative
- [ ] Figure captions are self-contained
- [ ] Tables are properly formatted
- [ ] Table captions are self-contained
- [ ] Results are presented objectively without interpretation
- [ ] Statistical significance is assessed appropriately

### Discussion

- [ ] Results are interpreted in context of literature
- [ ] Limitations are acknowledged
- [ ] Alternative interpretations are considered
- [ ] Future work is suggested
- [ ] Conclusions are supported by the data
- [ ] Overstated claims are avoided

### Figures

- [ ] All figures are referenced in the text
- [ ] Figure quality is suitable for publication (≥ 300 dpi)
- [ ] Figure axes are labeled with units
- [ ] Figure legends are self-contained
- [ ] Color figures are color-blind friendly
- [ ] Error bars are shown where appropriate
- [ ] Statistical significance is indicated

### Tables

- [ ] All tables are referenced in the text
- [ ] Table format is consistent
- [ ] Table columns are labeled with units
- [ ] Table captions are self-contained
- [ ] Uncertainties are reported in parentheses
- [ ] Significant figures are appropriate

### References

- [ ] All references are cited in the text
- [ ] All citations have corresponding references
- [ ] Reference format is consistent
- [ ] DOIs are included where available
- [ ] ArXiv identifiers are included for preprints
- [ ] References are complete and accurate

### Data and Code

- [ ] Data availability statement is included
- [ ] Code availability statement is included
- [ ] Data are deposited in a public repository
- [ ] Code is deposited in a public repository
- [ ] Data and code are properly cited
- [ ] Reproducibility guide is provided

### Ethics

- [ ] No plagiarism
- [ ] No self-plagiarism
- [ ] No duplicate publication
- [ ] No undisclosed conflicts of interest
- [ ] Human/animal subjects (if applicable) have IRB/IACUC approval
- [ ] Data use agreements are followed

## Journal-Specific Requirements

### AAS Journals (ApJ, AJ, ApJS)

- [ ] Manuscript follows AAS Journal Guide for Authors
- [ ] Uses AASTeX format
- [ ] Figures are in PDF or EPS format
- [ ] Table format is correct
- [ ] References are in AAS format
- [ ] Data availability statement is included
- [ ] Software citation is included

### MNRAS

- [ ] Manuscript follows MNRAS Instructions for Authors
- [ ] Uses MNRAS LaTeX template
- [ ] Figures are in high-resolution format
- [ ] Table format is correct
- [ ] References are in MNRAS format
- [ ] Data availability statement is included

### Nature Astronomy

- [ ] Abstract is ≤ 150 words
- [ ] Methods section is concise
- [ ] Extended methods are in Supplementary Information
- [ ] Figures are in high-resolution format
- [ ] Data availability statement is included
- [ ] Competing interests statement is included

## Common Issues to Avoid

### Overstated Claims

- [ ] Avoid "ruling out" alternatives without statistical evidence
- [ ] Avoid definitive claims about artificial origins
- [ ] Use cautious language ("constraining" instead of "ruling out")
- [ ] Acknowledge alternative interpretations

### Missing Uncertainties

- [ ] All numerical values include uncertainties
- [ ] Uncertainties are propagated through calculations
- [ ] Error bars are shown on figures
- [ ] Confidence intervals are reported

### Missing Methodology Details

- [ ] All selection criteria are justified
- [ ] All statistical methods are described
- [ ] All ML training procedures are documented
- [ ] All assumptions are stated

### Missing References

- [ ] All methods are properly referenced
- [ ] All data sources are cited
- [ ] All software packages are cited
- [ ] All models are cited

## Submission Checklist

### Pre-Submission

- [ ] Read journal's Guide for Authors
- [ ] Check word count limits
- [ ] Check figure/table limits
- [ ] Check reference limits
- [ ] Prepare cover letter
- [ ] Prepare suggested reviewers
- [ ] Prepare disclosure statements

### Submission Files

- [ ] Main manuscript file
- [ ] Figure files (high resolution)
- [ ] Table files
- [ ] Supplementary materials
- [ ] Cover letter
- [ ] Conflict of interest statement
- [ ] Data availability statement
- [ ] Code availability statement

## Post-Submission

### Peer Review

- [ ] Respond to all reviewer comments
- [ ] Revise manuscript accordingly
- [ ] Provide point-by-point response
- [ ] Highlight changes in revised manuscript
- [ ] Submit revised manuscript

### After Acceptance

- [ ] Sign copyright transfer agreement
- [ ] Pay publication fees
- [ ] Submit final version
- [ ] Update arXiv version
- [ ] Update data/code repositories
